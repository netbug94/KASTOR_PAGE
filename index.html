Criptografía de Bóveda

Nuestra arquitectura de seguridad entrelaza tus datos con algoritmos avanzados. Las llaves nunca tocan el servidor; el descifrado ocurre en el lado del cliente, garantizando que tu información permanezca completamente privada.

Flujo de Seguridad Zero-Knowledge

Tus llaves maestras nunca cruzan la red.

1

Derivación de Llave Local (KEK)

Tu contraseña maestra se combina con un salt aleatorio local. Se deriva una Key Encryption Key (KEK) de 32 bytes usando PBKDF2-HMAC-SHA256 con 600,000 iteraciones en tu dispositivo.

2

Cifrado de Llave de Datos (DEK)

Se genera una llave de datos de 256 bits (DEK) para encriptar los parámetros de tus conexiones. Esta llave se cifra con la KEK antes de enviarse de forma segura a la base de datos.

3

Protección de Datos con AES-GCM

Cada url y token se encripta de forma local en tu navegador con AES-256-GCM usando la DEK. El servidor sólo almacena paquetes cifrados ilegibles.

🪵

Derivación Resistente

El uso de PBKDF2 de alta iteración imposibilita ataques de fuerza bruta en el servidor, garantizando seguridad absoluta frente a filtraciones de datos.
