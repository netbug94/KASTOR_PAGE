Zero-Knowledge System Architecture

Your master keys never cross the network boundary.

1

Client-Side KEK Derivation

Your master password is combined with a random salt locally. A 32-byte Key Encryption Key (KEK) is derived using the PBKDF2-HMAC-SHA256 algorithm with 600,000 iterations.

2

Data Encryption Key (DEK) Wrapping

A random 256-bit Data Encryption Key (DEK) is generated to encrypt connection slots. The DEK itself is encrypted (wrapped) using the KEK before being sent to the database.

3

AES-GCM Payload Protection

Every individual slot URL and token parameter is encrypted locally using the DEK with AES-256-GCM. The server stores only raw ciphertext envelopes.

🔑

Strong Key Derivation

Utilizes PBKDF2 with SHA-256 and 600k iterations to derive key encryption keys, preventing brute force attacks even in the case of a complete database compromise.

📦

26 Isolated Slots (A-Z)

Supports up to 26 independent parameter slots containing secure endpoint URLs and authorization keys, allowing modular management of environments.

📱

Cross-Platform Design

Uses standardized cryptographic primitives, ensuring complete compatibility between desktop web portals and native mobile clients.
